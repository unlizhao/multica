模块全景与深度解析

本目录给出 Multica 的全模块清单(技术 + 业务),并对其中已沉淀深度解析的模块给出独立解析页。 模块按"核心 / 重要 / 一般"三档标记重要程度;重点深度解析 部分对核心技术对象逐一展开。

★★★ 核心 ★★ 重要 ★ 一般 · 文件名为 路径/file.ext 表示主入口源码位置

重点深度解析(技术对象 · 7 篇)

这 7 个对象覆盖"前后端契约 · 应用装配 · 任务生命周期"三条主干,共 12 张 SVG 图。读完即理解 Multica 的核心运作方式。

01
ApiClient
packages/core/api/client.ts

前端唯一 HTTP 入口。统一 header、CSRF、请求 ID、401 处理、ApiError 结构化错误。
02
parseWithFallback
packages/core/api/schema.ts

API 响应契约的最后防线。zod 校验失败只 warn 不 throw,装机版 Electron 不会白屏。
03
WSClient × realtime.Hub
core/api/ws-client.ts · server/internal/realtime/hub.go

前后端 WebSocket 双方:订阅/广播协议、自动重连、scope room、Redis 多节点中继与去重。
04
CoreProvider
packages/core/platform/core-provider.tsx

应用启动装配的"主板"。一次性创建 ApiClient、stores、QueryClient、WS、I18n,Web/Desktop 都包它。
05
setCurrentWorkspace
packages/core/platform/workspace-storage.ts

"当前工作区"的全局单例。微任务防 React 19 跨组件更新;命名空间存储防止跨工作区数据泄漏。
06
TaskService
server/internal/service/task.go

Agent 任务全生命周期(enqueue → claim → start → complete/fail → retry)。Multica 业务最重的一段代码。
07
AuthStore
packages/core/auth/store.ts

同时支持 Web 的 cookieAuth 与 Desktop 的 token 模式。401 与瞬时错误的区分,析构时清理工作区单例。
